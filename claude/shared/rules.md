# Rules

## General
- Always read task before implementation
- Do not overengineer
- MVP first, scale later

## Backend
- Keep endpoints minimal
- Validate inputs
- Handle errors clearly

## Frontend
- Simple UI
- Fast interaction
- No complex state management

## Prompt
- Always structured output
- Deterministic format
- No hallucinated fields

## SEO
- Focus on high intent keywords
- Avoid keyword stuffing

## Agents
- Do not assume missing info
- Always follow task file
